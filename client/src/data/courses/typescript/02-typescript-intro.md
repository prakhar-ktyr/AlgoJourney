# TypeScript Introduction

TypeScript is a typed superset of JavaScript that compiles to plain JavaScript.

## What is TypeScript?

TypeScript is an open-source programming language developed and maintained by **Microsoft**.

- It is a strict syntactical superset of JavaScript and adds optional static typing to the language.
- TypeScript is designed for the development of large applications and transpiles to JavaScript.
- As it is a superset of JavaScript, existing JavaScript programs are also valid TypeScript programs.

---

## Why Use TypeScript?

1. **Static Typing**: TypeScript's main feature is its static type system. You can define variables with a specific type, and the compiler will ensure they only ever hold values of that type. This catches bugs at compile-time rather than runtime.
2. **Better Tooling/IntelliSense**: IDEs and text editors provide much richer environment support (auto-completion, inline documentation) for TypeScript than they do for JavaScript.
3. **ECMAScript Features**: TypeScript supports the latest JavaScript features and allows you to compile them down to older JavaScript versions so they can run on all browsers.
4. **Large Codebases**: TypeScript shines in large projects, making it easier to refactor code and ensuring that changes in one part of the system don't unintentionally break another.

---

## TypeScript vs JavaScript

| Feature            | JavaScript             | TypeScript                 |
| ------------------ | ---------------------- | -------------------------- |
| **Typing**         | Dynamically Typed      | Statically Typed           |
| **Compilation**    | Interpreted at runtime | Transpiled into JavaScript |
| **Errors**         | Discovered at runtime  | Discovered at compile time |
| **Learning Curve** | Lower                  | Higher                     |
