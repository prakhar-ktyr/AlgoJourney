# React useId Hook

`useId` is a React Hook for generating unique IDs that can be passed to accessibility attributes.

It was introduced in React 18.

## The Problem

When building reusable form components, you often need to link an `<input>` to its `<label>` using the `id` and `htmlFor` attributes.

If you hardcode the ID in the component, and then use that component multiple times on the same page, you will end up with duplicate IDs. Duplicate IDs break accessibility (screen readers) and HTML validation.

```jsx
// Problem: If PasswordField is used twice, we have two elements with id="password"
function PasswordField() {
  return (
    <>
      <label htmlFor="password">Password:</label>
      <input type="password" id="password" />
    </>
  );
}
```

## The Solution: `useId`

`useId` generates a unique string that you can use for your IDs.

```jsx
import { useId } from 'react';

function PasswordField() {
  const passwordHintId = useId();

  return (
    <>
      <label htmlFor={passwordHintId}>Password:</label>
      <input type="password" id={passwordHintId} />
    </>
  );
}

function App() {
  return (
    <form>
      <h2>Choose Password</h2>
      <PasswordField />
      
      <h2>Confirm Password</h2>
      <PasswordField />
    </form>
  )
}
```

Now, even though `PasswordField` is rendered twice, React will ensure that the IDs generated by `useId` are unique across the entire application (e.g., `:r1:`, `:r2:`).

## Using `useId` with Multiple Elements

If you need to link multiple elements in the same component, you don't need to call `useId` multiple times. You can append a suffix to the base ID.

```jsx
import { useId } from 'react';

function FormField({ labelText }) {
  const id = useId();

  return (
    <div>
      <label htmlFor={id + '-input'}>{labelText}</label>
      <input type="text" id={id + '-input'} aria-describedby={id + '-hint'} />
      <p id={id + '-hint'}>Please enter your details here.</p>
    </div>
  );
}
```

## Important Note

**Do NOT use `useId` to generate keys in a list.**
Keys should be generated from your data (like a database ID). `useId` is strictly for generating unique identifiers for HTML attributes like `id`, `aria-describedby`, `aria-labelledby`, etc.
